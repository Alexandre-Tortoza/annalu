# Internationalization (i18n) Guide

## Overview

AnnaLu supports two languages: **Portuguese (PT)** and **English (EN)**.

- **Portuguese** is the default locale and appears at the root (`/`, `/galeria`, `/artwork/slug`)
- **English** is a secondary locale at the `/en/` prefix (`/en/`, `/en/gallery`, `/en/art/slug`)

The i18n system is built on Astro's built-in `i18n` routing configuration plus a custom translation module.

---

## Quick Reference

### Using Translations in a Component

```astro
---
import { useTranslations } from '@/i18n'

interface Props {
  lang: 'pt' | 'en'
}

const { lang } = Astro.props
const t = useTranslations(lang)
---

<h1>{t('nav.home')}</h1>
<button>{t('hero.cta')}</button>
```

### Translation Helper Functions

```ts
// Get translations for a locale
const t = useTranslations('pt')
t('nav.home')  // → 'Inicio'

// Detect locale from URL
const lang = getLangFromUrl(Astro.url)  // → 'pt' or 'en'

// Get alternate URL (switches language)
const enUrl = getAlternateUrl(Astro.url, 'en')
// '/galeria' → '/en/gallery'
// '/artwork/slug' → '/en/art/slug'
```

## How It Works

### 1. Routing (Astro i18n)

Configured in `astro.config.mjs`:

```ts
i18n: {
  defaultLocale: 'pt',
  locales: ['pt', 'en'],
  routing: { prefixDefaultLocale: false }
}
```

This tells Astro to:
- Treat `pt` as the default locale (appears at root)
- Treat `en` as a secondary locale (prefixed with `/en/`)
- Generate both `hreflang` links and sitemap entries

### 2. UI Strings (translations.ts)

All user-facing strings are centralized in `src/i18n/translations.ts`:

```ts
export const translations = {
  pt: {
    'nav.home': 'Início',
    'nav.gallery': 'Galeria',
    'hero.title': 'A sua galeria de arte online',
    'hero.cta': 'Ver Galeria',
    'gallery.title': 'Galeria',
    'gallery.description': 'Explore todas as obras...',
    'section.latest': 'Últimas Obras',
    'section.about': 'Sobre a Artista',
    'footer.copyright': '© 2024 AnnaLu. Todos os direitos reservados.',
    'aria.toggleTheme': 'Alternar tema',
    'aria.languagePicker': 'Selecionar idioma'
  },
  en: {
    'nav.home': 'Home',
    'nav.gallery': 'Gallery',
    'hero.title': 'Your online art gallery',
    'hero.cta': 'View Gallery',
    'gallery.title': 'Gallery',
    'gallery.description': 'Explore all artworks...',
    'section.latest': 'Latest Works',
    'section.about': 'About the Artist',
    'footer.copyright': '© 2024 AnnaLu. All rights reserved.',
    'aria.toggleTheme': 'Toggle theme',
    'aria.languagePicker': 'Select language'
  }
}
```

### 3. Content Fields (bilingual schema)

Artwork content has bilingual fields:

```yaml
title: "Masterpiece"          # English
titlePt: "Obra Prima"         # Portuguese
description: "English desc"   # English
descriptionPt: "Descrição"    # Portuguese
```

## Adding New UI Strings (Step-by-Step Example)

### Scenario: Add a "View Details" button to artwork cards

#### Step 1: Add to translations.ts

Open `src/i18n/translations.ts`:

```ts
export const translations = {
  pt: {
    // ... existing strings ...
    'card.viewDetails': 'Ver Detalhes',  // ← Add here
  },
  en: {
    // ... existing strings ...
    'card.viewDetails': 'View Details',  // ← Add here too!
  }
} as const;
```

**Key points:**
- Always add to **both** `pt` and `en` objects
- Use semantic keys: `namespace.action` (e.g., `card.`, `nav.`, `button.`)
- Both strings must have the same key

#### Step 2: Use in Your Component

In `src/components/ArtworkCard.astro`:

```astro
---
import { useTranslations } from '@/i18n'

interface Props {
  lang: 'pt' | 'en'
  artwork: CollectionEntry<'artworks'>
}

const { lang, artwork } = Astro.props
const t = useTranslations(lang)  // ← Initialize translations helper
---

<article class="artwork-card">
  <h2>{lang === 'pt' ? artwork.data.titlePt : artwork.data.title}</h2>
  <img src={artwork.data.cover.src} alt={artwork.data.cover.alt} />
  <button>
    {t('card.viewDetails')}  {/* ← Use translation */}
  </button>
</article>
```

#### Step 3: TypeScript Benefits

The `useTranslations()` function is **type-safe**:

```ts
const t = useTranslations('pt')
t('card.viewDetails')    // ✅ IDE auto-completes, no errors
t('nonexistent.key')     // ⚠️ Returns undefined, no compile error (watch for typos!)
```

To catch typos, run:
```bash
pnpm astro check
```

## Content vs UI Strings

### Content Strings (Bilingual in Schema)

Artwork titles, descriptions, and metadata are **part of the content**. Store them in markdown files with both PT and EN versions:

**File:** `src/content/artworks/masterpiece.md`
```yaml
---
title: "Masterpiece"
titlePt: "Obra Prima"
description: "An abstract acrylic composition"
descriptionPt: "Uma composição acrílica abstrata"
---
```

**In component:**
```astro
<h1>
  {lang === 'pt' ? artwork.data.titlePt : artwork.data.title}
</h1>
```

### UI Strings (Translations.ts)

All buttons, labels, nav items, and static text go in `translations.ts`:

**File:** `src/i18n/translations.ts`
```ts
export const translations = {
  pt: {
    'nav.home': 'Inicio',
    'nav.gallery': 'Galeria',
  },
  en: {
    'nav.home': 'Home',
    'nav.gallery': 'Gallery',
  }
}
```

**In component:**
```astro
<button>{t('nav.home')}</button>
```

## Routing Map (URL Pairs)

When users switch languages, URLs change according to this map:

| Portuguese Path | English Path | Example |
|---|---|---|
| `/` | `/en/` | Home page |
| `/galeria` | `/en/gallery` | Gallery listing |
| `/artwork/sunset` | `/en/art/sunset` | Artwork detail |
| `/artwork/abstract-blue` | `/en/art/abstract-blue` | Artwork detail |

The mapping is defined in `src/i18n/index.ts`:

```ts
const routeMap: Record<string, string> = {
  '/galeria': '/gallery',
  '/artwork/': '/art/',
}
```

## Common i18n Patterns & Gotchas

### ✅ Do: Keep Keys Consistent

```ts
// Good: semantic naming with namespaces
'nav.home', 'nav.gallery', 'nav.language'
'button.submit', 'button.cancel'
'aria.toggleTheme', 'aria.openMenu'
```

### ❌ Don't: Hardcode Text

```astro
<!-- ❌ Wrong -->
<h1>Welcome</h1>

<!-- ✅ Correct -->
<h1>{t('hero.title')}</h1>
```

### ✅ Do: Test Both Languages

After adding translations, test:
```bash
pnpm dev
```

1. Visit `/` and check PT text renders
2. Visit `/en/` and check EN text renders
3. Switch language with language picker
4. Verify hreflang links in page source (`Ctrl+U`)

### ❌ Don't: Forget the Other Language

```ts
export const translations = {
  pt: {
    'new.string': 'Portuguese text',  // ✅ Added
  },
  en: {
    'new.string': 'English text',     // ⚠️ IMPORTANT: Add to EN too!
  }
}
```

### ✅ Do: Conditional Pluralization

```ts
export const translations = {
  pt: {
    'count.singular': 'obra',
    'count.plural': 'obras',
  },
  en: {
    'count.singular': 'artwork',
    'count.plural': 'artworks',
  }
}
```

Then use:
```astro
<p>
  {count} {count === 1 ? t('count.singular') : t('count.plural')}
</p>
```

## Page Structure

### Portuguese Pages

Located at `src/pages/` (root):

```
src/pages/
├── index.astro                    → /
├── galeria.astro                  → /galeria
└── artwork/
    └── [slug].astro               → /artwork/slug
```

### English Pages

Located at `src/pages/en/` (prefixed):

```
src/pages/en/
├── index.astro                    → /en/
├── gallery.astro                  → /en/gallery
└── art/
    └── [slug].astro               → /en/art/slug
```

### Page Implementation

**Portuguese page** (`src/pages/index.astro`):
```astro
---
import BaseLayout from '../layouts/BaseLayout.astro'
import { useTranslations } from '../i18n'

const lang = 'pt'
const t = useTranslations(lang)
const alternateUrl = '/en/'
---

<BaseLayout seo={...} lang={lang} alternateUrl={alternateUrl}>
  <h1>{t('hero.title')}</h1>
</BaseLayout>
```

**English page** (`src/pages/en/index.astro`):
```astro
---
import BaseLayout from '../../layouts/BaseLayout.astro'
import { useTranslations } from '../../i18n'

const lang = 'en'
const t = useTranslations(lang)
const alternateUrl = '/'
---

<BaseLayout seo={...} lang={lang} alternateUrl={alternateUrl}>
  <h1>{t('hero.title')}</h1>
</BaseLayout>
```

## URL Switching (getAlternateUrl)

When users click the language picker, the URL must change according to the route mapping.

### Example: Switching from PT to EN

```ts
// User is on Portuguese gallery page
getAlternateUrl(new URL('https://annalu.art/galeria'), 'en')
// → 'https://annalu.art/en/gallery'

// User is on English gallery page
getAlternateUrl(new URL('https://annalu.art/en/gallery'), 'pt')
// → 'https://annalu.art/galeria'
```

This is handled automatically by the `LanguagePicker` component.

## SEO & hreflang

Every page includes `<link rel="alternate" hreflang>` tags:

```html
<link rel="alternate" hreflang="pt" href="https://annalu.art/artwork/slug">
<link rel="alternate" hreflang="en" href="https://annalu.art/en/art/slug">
<link rel="alternate" hreflang="x-default" href="https://annalu.art/artwork/slug">
```

This is handled automatically by the `SEO` component:

```astro
---
import SEO from '../components/SEO.astro'

const lang = 'pt'
const alternateUrl = '/en/artwork/slug'
---

<SEO
  title="Artwork Title"
  description="..."
  canonical="https://annalu.art/artwork/slug"
  lang={lang}
  alternateUrl={alternateUrl}
/>
```

## Locale Detection

### From URL

```ts
import { getLangFromUrl } from '@/i18n'

const locale = getLangFromUrl(Astro.url)
// '/galeria' → 'pt'
// '/en/gallery' → 'en'
// '/' → 'pt' (default)
```

### In Components (Reusable)

Use in any component that needs to detect the current language from the URL:

```astro
---
import { getLangFromUrl } from '@/i18n'

const lang = getLangFromUrl(Astro.url)  // Auto-detects from current URL
const t = useTranslations(lang)
---

<p>{t('nav.home')}</p>
```

### In Pages (Hard-coded)

Each language has its own pages, so hard-code the locale:

```astro
---
// In src/pages/index.astro (Portuguese page)
const lang = 'pt'
const t = useTranslations(lang)
---
```

```astro
---
// In src/pages/en/index.astro (English page)
const lang = 'en'
const t = useTranslations(lang)
---
```

## More i18n Patterns

### Language Picker in Header

The `LanguagePicker` component automatically switches between languages:

```astro
---
import LanguagePicker from '@/components/LanguagePicker.astro'
import { getAlternateUrl } from '@/i18n'

const lang = 'pt'
const alternateUrl = getAlternateUrl(Astro.url, 'en')
---

<header>
  <LanguagePicker lang={lang} alternateUrl={alternateUrl} />
</header>
```

### Conditional Translation (Plurals)

```astro
---
const t = useTranslations(lang)

const countText = artworks.length === 1
  ? t('gallery.count')      // 'obra'
  : t('gallery.count')      // 'obras' (handle in translation)
---

<p>{artworks.length} {countText}</p>
```

### Bilingual Meta Tags

```astro
---
const title = lang === 'pt'
  ? artwork.data.titlePt
  : artwork.data.title

const description = lang === 'pt'
  ? artwork.data.descriptionPt
  : artwork.data.description
---

<title>{title} | AnnaLu</title>
<meta name="description" content={description} />
```

## Migration Guide: Adding a New Language (Future)

To add a third language (e.g., Spanish):

### 1. Update astro.config.mjs

```ts
i18n: {
  defaultLocale: 'pt',
  locales: ['pt', 'en', 'es'],
  routing: { prefixDefaultLocale: false }
}
```

### 2. Add translations

```ts
export const translations = {
  pt: { ... },
  en: { ... },
  es: { ... }  // Add Spanish translations
}
```

### 3. Create `/es/` pages

```
src/pages/es/
├── index.astro
├── gallery.astro
└── art/[slug].astro
```

### 4. Update route mapping

```ts
const routeMap = {
  '/galeria': '/gallery',
  '/artwork/': '/art/',
  '/galeria': '/galeria-es',  // Spanish variant
}
```

## Troubleshooting

### Pages Not Rendering in EN

1. Verify `src/pages/en/` directory exists
2. Check filenames match PT structure with adjusted paths:
   - `index.astro` (both)
   - `galeria.astro` → `gallery.astro`
   - `artwork/[slug].astro` → `art/[slug].astro`

### hreflang Links Incorrect

1. Run `pnpm build`
2. Check `dist/sitemap-0.xml` for correct hreflang entries
3. Verify `getAlternateUrl()` route mapping is correct

### Translations Not Appearing

1. Verify key exists in both `pt` and `en` in `translations.ts`
2. Check component is using `useTranslations(lang)` correctly
3. Run `pnpm astro check` to catch undefined translations

### Mixed Language Content

Content should be bilingual (PT + EN fields), not split by file.

❌ **Don't:**
```
artworks/
├── pt/
│   └── artwork.md
└── en/
    └── artwork.md
```

✅ **Do:**
```
artworks/
└── artwork.md
   (contains title, titlePt, description, descriptionPt)
```

## Best Practices

✅ **Do:**
- Keep translation keys flat and semantic (`nav.home`, `gallery.title`)
- Update both PT and EN translations together
- Test both language versions after making changes
- Use `lang` prop consistently in components
- Run `pnpm astro check` to validate TypeScript
- Use bilingual content schema for artwork metadata

❌ **Don't:**
- Hardcode text in components (always use `t()`)
- Create separate content files per language (use bilingual schema)
- Forget to update the other language when adding strings
- Auto-translate without human review
- Use HTML tags in translation strings (move to component)
